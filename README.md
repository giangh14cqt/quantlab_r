# QuantLab — High-Speed Strategy Backtester

QuantLab is a high-performance R package designed for rapid strategy simulation and backtesting on historical market data. It uniquely combines the flexibility of **R6 object-oriented programming**, the safety of an **S4 validation layer**, and the speed of a compiled **C++ engine** (via `Rcpp`) for fast daily simulation loops.

---

## Architecture & Core Design

QuantLab is split into three main decoupled layers:

1. **Strategy Layer (R6)**: Handles signal generation. Strategies (such as rolling maximum breakout or DCA interval matching) generate signals using vectorized operations in R.
2. **Configuration Layer (S4)**: Enforces input parameter validity. S4 validation constraints ensure configurations (like lookback periods or threshold fractions) are checked prior to execution.
3. **Execution Engine (C++ / Rcpp)**: Executes the daily path-dependent simulation loop. It computes daily portfolios, manages cash flow, processes fractional shares, and applies stop-loss exits.


## Simulation Models

QuantLab supports two distinct simulation models depending on configuration parameters:

### 1. All-In Breakout Model (`invest_amount = 0`)
* **Core Logic**: Deploys the entire initial cash balance (`initial_cash`) on the first strategy signal.
* **Stop-Loss Protection**: If the asset price drops below the specified threshold (`stop_loss`), the engine triggers a stop-loss sell at the day's low price and returns the capital to cash.
* **Best Fit**: Breakout and momentum strategies (e.g., `ATH`).

### 2. Systematic Accumulation / Fresh-Money Model (`invest_amount > 0`)
* **Core Logic**: Simulates a savings plan where a fixed amount (`invest_amount`) of new money is injected on every buy signal. 
* **Cash Flow**: The investor's total capital increases over time as new money is injected (no cash exhaustion).
* **Best Fit**: Dollar-cost averaging and dip buying (e.g., `DCA`, `Dip`).

## Advanced Performance Metrics

Standard investment metrics (like classic CAGR or naive Sharpe Ratio) break down when cash is continuously injected:
* **Personal Return**: Calculated as `(Final Portfolio Value - Total Capital Invested) / Total Capital Invested`, ensuring a correct base for systematic savings.
* **Annualized Return (CAGR)**: Uses the total capital invested as the base, annualizing the personal return over the duration of the backtest.
* **Injection-Adjusted Sharpe Ratio**: Classic Sharpe ratio daily return calculations count cash deposits as market gains. QuantLab adjusts daily returns to exclude deposits:
  $$\Delta r_t = \frac{(E_t - E_{t-1}) - \Delta CI_t}{E_{t-1}}$$
  This removes deposit spikes, calculating the true volatility and returns generated by the asset alone.

## Getting Started

### Prerequisites
* R (>= 4.0.0)
* RStudio (recommended)
* C++ compiler (Rtools on Windows, Xcode Command Line Tools on macOS, or GCC on Linux)

### Installation

Clone the repository and open `quantlab_r.Rproj` in RStudio, then run:

```r
# Install development dependencies and build the package
devtools::install()
```

### Basic Usage

#### 1. Running an All-In Breakout Strategy (ATH)
```r
library(quantlab)

# Load bundled WIG index data (includes Polish header mapping)
path <- system.file("extdata", "wig_d.csv", package = "quantlab")
data <- load_market_data(path)

# Configure ATH breakout strategy with a 252-day window and a 10% stop-loss
cfg_ath <- strategy_config("ATH", lookback = 252L, stop_loss = 0.10, initial_cash = 10000)

# Run simulation
res_ath <- run_backtest(data, cfg_ath)
print(res_ath)
plot(res_ath)
```

#### 2. Running a Systematic Accumulation Strategy (DCA & Dip)
```r
# DCA: invest 500 PLN every 21 trading days
cfg_dca <- strategy_config("DCA", invest_amount = 500, dca_interval = 21L)
res_dca <- run_backtest(data, cfg_dca)
summary(res_dca)

# Buy-the-Dip: invest 1000 PLN whenever price drops 10% below the running all-time high
cfg_dip <- strategy_config("Dip", dip_pct = 0.10, invest_amount = 1000)
res_dip <- run_backtest(data, cfg_dip)
summary(res_dip)
```

#### 3. Launching the Interactive Dashboard
Compare all three strategies visually inside a premium Shiny interface:
```r
launch_app()
```

## Project Structure

```
├── R/
│   ├── strategy.R          — Strategy abstract R6 base class and subclasses (ATH, DCA, Dip)
│   ├── strategy_config.R   — Formally validated S4 StrategyConfig class
│   ├── run_backtest.R      — End-to-end backtesting pipeline entry point and launch_app()
│   ├── metrics.R           — Vectorized performance metrics (Sharpe ratio, Max Drawdown)
│   └── data_loader.R       — Market data ingestion with Polish header mappings (stooq.pl)
├── src/
│   └── backtest_engine.cpp — Compiled C++ simulation loop for path-dependent updates (Rcpp)
├── inst/
│   ├── extdata/            — Bundled index CSV datasets (WIG, S&P 500, DAX)
│   └── shiny_app/app.R     — Interactive dashboard UI/Server comparing DCA, Dip, and ATH
├── tests/
│   └── testthat/           — Unit tests validating simulation engine modes and strategy signals
└── man/                    — Auto-generated package documentation (roxygen2)
```
